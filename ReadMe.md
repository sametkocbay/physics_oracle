# Dataset Generation Guide
## RANS k-ω SST Initialization for 2D Airfoil Flow

This document is the implementation reference for generating the CFD dataset using Claude Code. It covers sampling, meshing, OpenFOAM setup, field extraction, and quality control. Follow the sections in order.

---

## 1. Scope and Operating Envelope

- **Geometry family:** NACA 4-digit profiles (primary), optionally NACA 5-digit (secondary)
- **Dimensionality:** 2D
- **Flow regime:** Steady, incompressible, fully turbulent (no transition modeling)
- **Turbulence model:** k-ω SST
- **AoA range:** -5° to +15° (attached and mildly separated; avoid deep stall)
- **Reynolds number range:** 1×10⁵ to 1×10⁷ (log-spaced)
- **Mach:** Incompressible assumption valid (M < 0.3)

Any case outside this envelope is an "OOD probe" — store separately, never use for training or hyperparameter tuning.

---

## 2. Sampling Strategy

Do **not** use a uniform grid over the parameter space. Use Latin Hypercube or Sobol sampling throughout.

### 2.1 Geometry sampling
- NACA 4-digit parameters: max camber (0–6%), camber position (20–60% chord), thickness (8–18%)
- Sample the joint (camber, position, thickness) space with Latin Hypercube
- Target: ~100–150 distinct profiles for training, ~30 for validation, ~30 for test

### 2.2 Flow condition sampling
- AoA: linear sampling in [-5°, +15°]
- Re: **log-uniform** in [1e5, 1e7] — never linear
- Use Latin Hypercube over the joint (profile_id, AoA, log Re) space
- Target: ~500 total cases

### 2.3 Dataset splits

| Split | Profiles | Purpose |
|-------|----------|---------|
| Train | 70% of profiles | Model training |
| Val | 15% of profiles (unseen) | Hyperparameter tuning, early stopping |
| Test | 15% of profiles (unseen) | Final reporting |
| OOD probe | Atypical profiles + extreme conditions | Reporting only |

OOD probe should include: AoA > 15°, Re < 1e5, Re > 1e7, and a few non-NACA profiles.

---

## 3. Case Naming Convention

```
[NACA_CODE]_[AoA]_[Re]
```

**Format rules:**
- `NACA_CODE`: 4-digit NACA code, e.g. `2412`, `0012`, `4415`
- `AoA`: signed, one decimal place, `p` for positive, `n` for negative (filesystem-safe)
  - `+5.0°` → `p5.0`, `-2.5°` → `n2.5`, `0.0°` → `p0.0`
- `Re`: scientific notation, e.g. `1.5e6`, `3.0e5`

**Examples:**
- `NACA2412_p5.0_1.5e6`
- `NACA0012_n2.5_3.0e5`
- `NACA4415_p10.0_5.0e6`

## 4. Mesh Generation

Use a **deformed C-grid**: same topology for every case, deformed to fit each airfoil. This is critical for keeping dataset size manageable and avoiding mesh variability as a confounding factor in training.

### 4.1 Tooling
- Gmsh with a parameterized C-grid template, or `blockMesh` with a morphing script
- The mesh script must be fully deterministic: same airfoil coordinates → same mesh, always

### 4.2 Quality requirements
- **y+ < 1** at the wall (k-ω SST low-Re mode, resolves viscous sublayer)
- **Growth ratio < 1.2** in the boundary layer normal direction
- **≥ 30 cells** in the boundary layer
- **Far field ≥ 20 chord lengths** from the airfoil surface
- Run `checkMesh`; reject if non-orthogonality > 70 or skewness > 4

## 5. OpenFOAM Case Setup

Each case lives inside its named directory under `dataset/cases/`. The OpenFOAM case is stored at `of_case/` within the case directory (see Section 8 for full layout).

### 5.1 of_case/ directory structure

```
of_case/
├── 0/
│   ├── U
│   ├── p
│   ├── k
│   ├── omega
│   └── nut
├── constant/
│   ├── polyMesh/          # generated by mesh script
│   └── turbulenceProperties
├── system/
│   ├── blockMeshDict      # or Gmsh equivalent
│   ├── controlDict
│   ├── fvSchemes
│   ├── fvSolution
│   └── sampleDict         # for post-processing extraction
└── postProcessing/        # written by OpenFOAM at runtime
```

### 5.2 Solver
- **Solver:** `simpleFoam` (steady-state incompressible RANS)
- **Turbulence model:** `kOmegaSST` set in `constant/turbulenceProperties`

`constant/turbulenceProperties`:
```
simulationType  RAS;
RAS { RASModel kOmegaSST; turbulence on; printCoeffs on; }
```

### 5.3 Boundary conditions (`0/`)

**`0/U`**
```
dimensions      [0 1 -1 0 0 0 0];
internalField   uniform (U_x U_y 0);   // rotated by AoA
boundaryField {
    inlet     { type fixedValue; value uniform (U_x U_y 0); }
    outlet    { type zeroGradient; }
    airfoil   { type noSlip; }
    top       { type slip; }
    bottom    { type slip; }
}
```

**`0/p`**
```
dimensions      [0 2 -2 0 0 0 0];
internalField   uniform 0;
boundaryField {
    inlet     { type zeroGradient; }
    outlet    { type fixedValue; value uniform 0; }
    airfoil   { type zeroGradient; }
    top       { type slip; }
    bottom    { type slip; }
}
```

**`0/k`**
- k_inlet = (3/2) · (|U| · I)²  with I = 0.01
```
dimensions      [0 2 -2 0 0 0 0];
internalField   uniform K_INLET;
boundaryField {
    inlet     { type fixedValue; value uniform K_INLET; }
    outlet    { type zeroGradient; }
    airfoil   { type kqRWallFunction; value uniform K_INLET; }
    top       { type slip; }
    bottom    { type slip; }
}
```

**`0/omega`**
- omega_inlet = k^(1/2) / (C_μ^(1/4) · L)  with C_μ = 0.09, L = 0.07 · chord
```
dimensions      [0 0 -1 0 0 0 0];
internalField   uniform OMEGA_INLET;
boundaryField {
    inlet     { type fixedValue; value uniform OMEGA_INLET; }
    outlet    { type zeroGradient; }
    airfoil   { type omegaWallFunction; value uniform OMEGA_INLET; }
    top       { type slip; }
    bottom    { type slip; }
}
```

**`0/nut`**
```
dimensions      [0 2 -1 0 0 0 0];
internalField   uniform 0;
boundaryField {
    inlet     { type calculated; value uniform 0; }
    outlet    { type calculated; value uniform 0; }
    airfoil   { type nutkWallFunction; value uniform 0; }
    top       { type slip; }
    bottom    { type slip; }
}
```

### 5.4 `system/controlDict`
```
application     simpleFoam;
startFrom       startTime;
startTime       0;
stopAt          endTime;
endTime         5000;
deltaT          1;
writeControl    timeStep;
writeInterval   500;
purgeWrite      1;              // keep only latest timestep + initial
writeFormat     ascii;
writePrecision  12;
runTimeModifiable true;

functions {
    forceCoeffs {
        type            forceCoeffs;
        libs            (forces);
        writeControl    timeStep;
        writeInterval   1;
        patches         (airfoil);
        rho             rhoInf;
        rhoInf          1;
        liftDir         (-sin(AoA_RAD) cos(AoA_RAD) 0);
        dragDir         (cos(AoA_RAD) sin(AoA_RAD) 0);
        CofR            (0.25 0 0);
        pitchAxis       (0 0 1);
        magUInf         U_MAG;
        lRef            1.0;
        Aref            1.0;
    }
    residuals {
        type            solverInfo;
        libs            (utilityFunctionObjects);
        writeControl    timeStep;
        writeInterval   1;
        fields          (U p k omega);
    }
}
```

### 5.5 `system/fvSchemes`
```
ddtSchemes      { default steadyState; }
gradSchemes     { default Gauss linear; }
divSchemes {
    default                         none;
    div(phi,U)                      Gauss linearUpwind grad(U);
    div(phi,k)                      Gauss linearUpwind grad(k);
    div(phi,omega)                  Gauss linearUpwind grad(omega);
    div((nuEff*dev(T(grad(U)))))    Gauss linear;
}
laplacianSchemes { default Gauss linear corrected; }
interpolationSchemes { default linear; }
snGradSchemes { default corrected; }
```

### 5.6 `system/fvSolution`
```
solvers {
    p {
        solver          GAMG;
        smoother        GaussSeidel;
        tolerance       1e-7;
        relTol          0.01;
    }
    U { solver smoothSolver; smoother GaussSeidel; tolerance 1e-8; relTol 0.1; }
    k { solver smoothSolver; smoother GaussSeidel; tolerance 1e-8; relTol 0.1; }
    omega { solver smoothSolver; smoother GaussSeidel; tolerance 1e-8; relTol 0.1; }
}
SIMPLE {
    nNonOrthogonalCorrectors 2;
    residualControl {
        p       1e-4;
        U       1e-5;
        k       1e-5;
        omega   1e-5;
    }
}
relaxationFactors {
    fields  { p 0.3; }
    equations { U 0.7; k 0.7; omega 0.7; }
}
```

### 5.7 Convergence criteria
Stop when **all** residuals have dropped ≥ 4 orders of magnitude from their initial value, OR force coefficients (Cl, Cd) change less than 0.1% over 200 consecutive iterations — whichever comes first. Maximum 5000 iterations; flag anything that hits the limit without converging.

## 6. Data to Store Per Case

After the simulation completes, extract and store the following. Use HDF5 (`.h5`) for array data and YAML for metadata.

### 6.1 `fields.h5` — converged solution
- `U` — velocity, (Ncells, 2)
- `p` — kinematic pressure, (Ncells,)
- `k` — turbulent kinetic energy, (Ncells,)
- `omega` — specific dissipation rate, (Ncells,)
- `nut` — turbulent viscosity, (Ncells,)
- `wall_distance` — distance to nearest wall, (Ncells,) — compute with `wallDist` utility

### 6.2 `mesh.h5` — geometry and connectivity
- `cell_centers` — (Ncells, 2)
- `points` — mesh vertex coordinates, (Npoints, 2)
- `connectivity` — cell-to-vertex mapping, (Ncells, Nvertices_per_cell)
- `boundary_markers` — integer label per cell: 0=interior, 1=wall, 2=inlet, 3=outlet, 4=farfield

### 6.3 `geometry.h5` — airfoil surface
- `airfoil_coordinates` — (N, 2), ordered from trailing edge, clockwise

### 6.4 `convergence.h5` — solver diagnostics
- `residual_history` — (Niter, 4) for [U, p, k, omega]
- `cl_history` — (Niter,)
- `cd_history` — (Niter,)
- `iterations_to_convergence` — scalar int
- `y_plus` — y+ values at wall-adjacent cell centers, (Nwall_cells,)
- `final_residuals` — dict of final values for U, p, k, omega

### 6.5 `meta.yaml`
```yaml
case_id: NACA2412_p5.0_1.5e6
naca_code: "2412"
aoa_deg: 5.0
Re: 1500000.0
U_inlet: [U_x, U_y]
U_mag: 15.0
nu: 1e-05
chord: 1.0
k_inlet: 3.375e-04
omega_inlet: 0.1234
mesh_version: "v1"
openfoam_version: "v2312"
solver_settings_hash: "abc123"   # md5 of fvSchemes + fvSolution
generation_timestamp: "2025-01-01T00:00:00Z"
converged: true
iterations_to_convergence: 1842
flags: []   # e.g. ["separated", "borderline_convergence"]
split: train
```

## 7. Quality Control

Run `scripts/quality_check.py` after each case. Log all rejections to `dataset/rejection_log.csv`.

| Check | Threshold | Action |
|-------|-----------|--------|
| Residuals dropped | < 4 orders | reject |
| Negative k anywhere in field | any | reject |
| Negative omega anywhere | any | reject |
| y+ at wall | > 5 | reject |
| Iterations | > 5000 | flag for manual review |
| Solver bounding warnings in log | any | flag |
| Cl/Cd physically unreasonable | case-dependent | flag for manual review |

Target acceptance rate: ~80–90%. Higher rejection rates indicate a mesh or BC problem.

## 9. Reproducibility Checklist

- [ ] OpenFOAM version pinned and recorded in `manifest.yaml`
- [ ] Mesh generation is deterministic: same airfoil input → same mesh
- [ ] Solver settings are identical across all cases (verified via `solver_settings_hash`)
- [ ] LHS/Sobol random seeds stored in `manifest.yaml`
- [ ] Splits stored as explicit case ID lists in `splits/`
- [ ] At least 5 cases re-run end-to-end and produce identical fields
- [ ] Rejection log preserved and non-empty
